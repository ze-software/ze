# Learned: spec-l2tp-11-web -- Web Management UI, CQM Graph, Disconnect

## What Was Built

L2TP web management UI at `/l2tp` with session list, detail page (CQM
graph via uPlot, event timeline, disconnect with audit trail), JSON/CSV/SSE
data feeds. Plus YANG grammar fix moving destructive commands to `clear`
verb, Service interface extension for observer access, and ObserverEvent
extension for disconnect audit (actor, reason, cause).

## Key Decisions

- **SSE per-connection ticker, not EventBroker.** CQM feeds are per-login;
  broadcasting would leak data and waste bandwidth. A `time.Ticker` at
  BucketInterval (100s) per SSE connection is simpler and naturally scoped.

- **Disconnect through CLI dispatch.** `clear l2tp session teardown <sid>
  reason <text> cause <code>` routes through the existing CommandDispatcher.
  Authz is enforced once at the CLI layer; the `clear` prefix is already
  denied in BuiltinReadOnlyProfile. Zero authz changes needed.

- **YANG grammar fix.** L2TP destructive commands moved from top-level
  `l2tp` noun to augmenting the `clear` verb. This aligns with ze's CLI
  grammar where verbs (show, clear, del, set) are top-level and nouns
  (bgp, l2tp, interface) are second-level.

- **Keyword arg parsing.** `reason <text...>` and `cause <code>` use
  keyword-prefixed grammar instead of positional args, making the command
  extensible without breaking existing callers.

## Mistakes / Surprises

- The original spec listed `l2tp session disconnect` as the command name,
  but the existing YANG had `l2tp session teardown`. Reusing the existing
  RPC handler avoided new YANG + handler code. The "disconnect" concept
  is a web UX term, not a protocol term.

- Five env vars (`ze.l2tp.enabled`, `ze.l2tp.shared-secret`, etc.) were
  documented as env var overrides but don't actually exist as registered
  env entries -- they're YANG config leaves only. The doc rewrite caught
  and removed them.

- Plugin config paths were wrong in initial docs (`plugins { l2tp-auth-local
  { ... } }` instead of `l2tp { auth { local { ... } } }`). The YANG
  schema uses the `l2tp` container as the root, not the plugin name.

## Patterns Worth Reusing

- **Service interface extension for observer.** Adding read methods
  (`SessionEvents`, `LoginSamples`) plus a write method (`RecordDisconnect`)
  to the existing Service interface kept one locator, one nil check, and
  no startup-ordering issues.

- **CSS custom properties for chart colors.** `--color-l2tp-established`
  etc. in `style.css`, read by JS via `getComputedStyle`. Theme changes
  propagate to charts automatically.

## Completion

All 11 L2TP child specs (1-wire through 11-web) are done. The umbrella
spec (l2tp-0) is marked done. The full BNG stack is implemented: wire
format, reliable delivery, tunnel/session FSMs, kernel integration, PPP
engine (LCP/auth/NCP), subsystem wiring, plugins (auth-local,
auth-radius, pool, shaper), observer/CQM, Prometheus metrics, and web UI.
