# RFC 6608: Subcodes for BGP Finite State Machine Error

## Meta

| Field | Value |
|-------|-------|
| RFC | 6608 |
| Title | Subcodes for BGP Finite State Machine Error |
| Status | Standards Track |
| Date | May 2012 |
| Updates | RFC 4271 |
| Errata | None |

**Purpose:** Defines subcodes for the BGP FSM Error (Error Code 5) to provide more diagnostic information about which state received an unexpected message.

## Constants

### FSM Error Subcodes (Error Code 5)

| Value | Name | Section |
|-------|------|---------|
| 0 | Unspecified Error | 3 |
| 1 | Receive Unexpected Message in OpenSent State | 3 |
| 2 | Receive Unexpected Message in OpenConfirm State | 3 |
| 3 | Receive Unexpected Message in Established State | 3 |

### NOTIFICATION Data Field

When sending FSM Error with subcodes 1-3, the Data field contains:

```
 0
 0 1 2 3 4 5 6 7
+-+-+-+-+-+-+-+-+
| Message Type  |
+-+-+-+-+-+-+-+-+
```

| Field | Size | Type | Description |
|-------|------|------|-------------|
| Message Type | 1 octet | uint8 | Type of the unexpected message received |

## MUST Requirements

### Tx: Sending NOTIFICATION

**Section 4:**

> "If a BGP speaker receives an unexpected message (e.g., KEEPALIVE/UPDATE/ROUTE-REFRESH message) on a session in OpenSent state, it MUST send to the neighbor a NOTIFICATION message with the Error Code Finite State Machine Error and the Error Subcode 'Receive Unexpected Message in OpenSent State'."

> "If a BGP speaker receives an unexpected message (e.g., OPEN/UPDATE/ROUTE-REFRESH message) on a session in OpenConfirm state, it MUST send a NOTIFICATION message with the Error Code Finite State Machine Error and the Error Subcode 'Receive Unexpected Message in OpenConfirm State' to the neighbor."

> "If a BGP speaker receives an unexpected message (e.g., OPEN message) on a session in Established State, it MUST send to the neighbor a NOTIFICATION message with the Error Code Finite State Machine Error and the Error Subcode 'Receive Unexpected Message in Established State'."

## Error Handling

| State | Unexpected Messages | Subcode | Data |
|-------|---------------------|---------|------|
| OpenSent | KEEPALIVE, UPDATE, ROUTE-REFRESH | 1 | Message type |
| OpenConfirm | OPEN, UPDATE, ROUTE-REFRESH | 2 | Message type |
| Established | OPEN | 3 | Message type |

**Note:** "Unexpected" means messages not valid for that FSM state per RFC 4271.

## State Machine Integration

| Current State | Valid Messages | Unexpected → Subcode |
|---------------|----------------|----------------------|
| OpenSent | OPEN, NOTIFICATION | All others → 1 |
| OpenConfirm | KEEPALIVE, NOTIFICATION | OPEN, UPDATE, ROUTE-REFRESH → 2 |
| Established | KEEPALIVE, UPDATE, NOTIFICATION, ROUTE-REFRESH | OPEN → 3 |

## Security Considerations

**Section 5:**

> "Specification, implementation, and deployment of the proposed BGP FSM Error subcodes could make BGP implementation fingerprinting easier and probably more accurate."

Operators should consider this when deciding whether to implement detailed subcodes.

## Implementation Notes

1. **Subcode 0 (Unspecified):** Use when the specific condition doesn't match subcodes 1-3, or for backwards compatibility
2. **Data field:** Always include the 1-octet message type for subcodes 1-3
3. **Message type values:** Per RFC 4271: OPEN=1, UPDATE=2, NOTIFICATION=3, KEEPALIVE=4, ROUTE-REFRESH=5
