# RFC 7606 - Revised Error Handling for BGP UPDATE Messages

## Meta

| Field | Value |
|-------|-------|
| RFC | 7606 |
| Title | Revised Error Handling for BGP UPDATE Messages |
| Status | Standards Track |
| Date | August 2015 |
| Updates | RFC 1997, 4271, 4360, 4456, 4760, 5543, 5701, 6368 |

**Purpose:** Minimize routing impact from malformed UPDATE messages by avoiding session resets when possible, instead using "treat-as-withdraw" or "attribute discard" approaches.

**Problem Solved:** The original BGP spec required session reset for any malformed attribute. This is problematic because:
- Affects all routes on the session, not just the bad one
- Optional transitive attributes may be "tunneled" through non-recognizing routers
- A single malformed attribute can fan out and reset many sessions

## Error-Handling Approaches

Listed from strongest to weakest action (Section 2):

| Approach | Action | When to Use |
|----------|--------|-------------|
| **Session reset** | Send NOTIFICATION, terminate session | Last resort; structural errors |
| **AFI/SAFI disable** | Ignore all subsequent routes for that AFI/SAFI | Per RFC 4760 Section 7 |
| **Treat-as-withdraw** | Process UPDATE as if routes were withdrawn | Preferred for most attribute errors |
| **Attribute discard** | Discard attribute, continue processing UPDATE | Only for attributes with no route selection effect |

### Critical Constraints

- **Treat-as-withdraw** requires successful parsing of NLRI field and/or MP_REACH/MP_UNREACH attributes
- **Attribute discard** MUST NOT be used for attributes affecting route selection
- When multiple errors exist with different approaches, use the **strongest** action

## Validation Rules

### UPDATE Message Structure (Section 3)

| Check | Valid | Invalid Action |
|-------|-------|----------------|
| Withdrawn Routes Length + Total Attribute Length + 23 > Message Length | No | Session reset (Malformed Attribute List) |
| MP_REACH_NLRI appears > 1 time | No | Session reset (Malformed Attribute List) |
| MP_UNREACH_NLRI appears > 1 time | No | Session reset (Malformed Attribute List) |
| Other attribute appears > 1 time | Yes | Discard all but first occurrence |
| Missing well-known mandatory attribute | No | Treat-as-withdraw |
| Attribute Flag O/T bits conflict with spec | No | Treat-as-withdraw (unless spec says otherwise) |

### Attribute Length Errors (Section 4)

| Check | Valid | Invalid Action |
|-------|-------|----------------|
| Last attribute length exceeds Total Attribute Length | No | Treat-as-withdraw |
| < 3 bytes remain (or < 4 if Extended Length) when parsing next attribute | No | Treat-as-withdraw |
| Attribute length = 0 (except AS_PATH, ATOMIC_AGGREGATE) | No | Treat-as-withdraw |

### NLRI Field Syntax (Section 5.3)

**IPv4 NLRI/Withdrawn Routes:**

| Check | Valid | Invalid Action |
|-------|-------|----------------|
| Any NLRI prefix length > 32 | No | Session reset (cannot parse NLRI) |
| Last NLRI length exceeds remaining data | No | Session reset (cannot parse NLRI) |

**MP_REACH_NLRI / MP_UNREACH_NLRI:**

| Check | Valid | Invalid Action |
|-------|-------|----------------|
| NLRI length inconsistent with AFI/SAFI (e.g., IPv4 > 32, IPv6 > 128) | No | AFI/SAFI disable or session reset |
| Last NLRI length exceeds remaining attribute data | No | AFI/SAFI disable or session reset |
| Attribute flags inconsistent with RFC 4760 | No | AFI/SAFI disable or session reset |
| MP_UNREACH_NLRI length < 3 | No | AFI/SAFI disable or session reset |
| MP_REACH_NLRI length < 5 | No | AFI/SAFI disable or session reset |
| MP_REACH Next Hop length inconsistent with AFI/SAFI | No | AFI/SAFI disable or session reset |

## Per-Attribute Error Handling (Section 7)

### Well-Known Mandatory Attributes

| Attribute | Type | Malformed If | Action |
|-----------|------|--------------|--------|
| ORIGIN (1) | Well-known mandatory | Length != 1 OR undefined value | Treat-as-withdraw |
| AS_PATH (2) | Well-known mandatory | See AS_PATH rules below | Treat-as-withdraw |
| NEXT_HOP (3) | Well-known mandatory | Length != 4 | Treat-as-withdraw |

### AS_PATH Malformation Rules (Section 7.2)

AS_PATH is malformed if any of:
- Unrecognized segment type
- Segment length would exceed attribute length (overrun)
- Only 1 byte remains after last segment (underrun - can't fit segment header)
- Segment has Path Segment Length = 0

**Leftmost AS check:** If implementation checks that leftmost AS = peer AS:
- SHOULD use treat-as-withdraw
- MAY use session reset if configured

### Well-Known Discretionary Attributes

| Attribute | Type | Malformed If | Action |
|-----------|------|--------------|--------|
| MULTI_EXIT_DISC (4) | Well-known discretionary | Length != 4 | Treat-as-withdraw |
| LOCAL_PREF (5) from EBGP | Well-known discretionary | Received from external peer | Attribute discard |
| LOCAL_PREF (5) from IBGP | Well-known discretionary | Length != 4 | Treat-as-withdraw |
| ATOMIC_AGGREGATE (6) | Well-known discretionary | Length != 0 | Attribute discard |
| AGGREGATOR (7) without 4-byte AS | Optional transitive | Length != 6 | Attribute discard |
| AGGREGATOR (7) with 4-byte AS | Optional transitive | Length != 8 | Attribute discard |

### Optional Attributes

| Attribute | Type | Malformed If | Action |
|-----------|------|--------------|--------|
| Community (8) | Optional transitive | Length = 0 OR length % 4 != 0 | Treat-as-withdraw |
| ORIGINATOR_ID (9) from EBGP | Optional non-transitive | Received from external peer | Attribute discard |
| ORIGINATOR_ID (9) from IBGP | Optional non-transitive | Length != 4 | Treat-as-withdraw |
| CLUSTER_LIST (10) from EBGP | Optional non-transitive | Received from external peer | Attribute discard |
| CLUSTER_LIST (10) from IBGP | Optional non-transitive | Length = 0 OR length % 4 != 0 | Treat-as-withdraw |
| MP_REACH_NLRI (14) | Optional non-transitive | See NLRI section | Session reset or AFI/SAFI disable |
| MP_UNREACH_NLRI (15) | Optional non-transitive | See NLRI section | Session reset or AFI/SAFI disable |
| Extended Community (16) | Optional transitive | Length = 0 OR length % 8 != 0 | Treat-as-withdraw |
| IPv6 Extended Community (25) | Optional transitive | Length = 0 OR length % 20 != 0 | Treat-as-withdraw |
| Traffic Engineering (24) | Optional transitive | Implementation-defined | Treat-as-withdraw |
| ATTR_SET (128) | Optional transitive | Implementation-defined | Treat-as-withdraw |

**Note:** Unrecognized Extended Community Type/Sub-Type MUST NOT be treated as error.

## MUST Requirements

### Tx (Sender)

- "The MP_REACH_NLRI or MP_UNREACH_NLRI attribute (if present) SHALL be encoded as the very first path attribute in an UPDATE message." (Section 5.1)
- "An UPDATE message MUST NOT contain more than one of the following: non-empty Withdrawn Routes field, non-empty Network Layer Reachability Information field, MP_REACH_NLRI attribute, and MP_UNREACH_NLRI attribute." (Section 5.1)

### Rx (Receiver)

- "Since older BGP speakers may not implement these restrictions, an implementation MUST still be prepared to receive these fields in any position or combination." (Section 5.1)
- "A BGP speaker advertising support for such a typed address family MUST handle routes with unrecognized NLRI types within that address family by discarding them." (Section 5.4)

### Validation

- Missing well-known mandatory attributes: "treat-as-withdraw" MUST be used (Section 3.d)
- Attribute Flag O/T conflicts: "treat-as-withdraw" approach MUST be used (Section 3.c)
- Multiple MP_REACH/MP_UNREACH: NOTIFICATION MUST be sent (Section 3.g)
- Multiple errors with same approach: specified approach MUST be used (Section 3.h)
- Multiple errors with different approaches: strongest action MUST be used (Section 3.h)

### Errors

- Withdrawn Routes Length + Total Attribute Length + 23 > Message Length: Error Subcode MUST be Malformed Attribute List (Section 3.b)
- UPDATE with path attributes but no reachable NLRI and any error other than attribute discard: "session reset" MUST be used (Section 5.2)

### State

- When treat-as-withdraw is used: routes MUST be removed from Adj-RIB-In per RFC 4271 (Section 2)

## SHOULD/MAY

- [SHOULD] Handle leftmost-AS check violations with treat-as-withdraw - maintains session stability
- [MAY] Use session reset for leftmost-AS check if configured - allows strict policy enforcement
- [MAY] Use AFI/SAFI disable per RFC 4760 Section 7 - alternative to session reset for MP errors

## Error Handling Decision Tree

```
1. Can NLRI be parsed?
   NO  --> Session reset (or AFI/SAFI disable for MP)
   YES --> Continue

2. Is it a structural error (length fields inconsistent)?
   YES --> Session reset
   NO  --> Continue

3. Is MP_REACH or MP_UNREACH duplicated?
   YES --> Session reset (Malformed Attribute List)
   NO  --> Continue

4. Is other attribute duplicated?
   YES --> Discard all but first, continue processing
   NO  --> Continue

5. Is well-known mandatory attribute missing?
   YES --> Treat-as-withdraw
   NO  --> Continue

6. Is attribute malformed?
   YES --> Use attribute-specific action from table
   NO  --> Process normally

7. Multiple attribute errors?
   Same approach for all --> Use that approach
   Different approaches --> Use strongest (session reset > AFI/SAFI disable > treat-as-withdraw > attribute discard)
```

## Special Cases

### UPDATE with Attributes but No NLRI (Section 5.2)

End-of-RIB (RFC 4724) can be:
- Empty UPDATE message
- UPDATE with only MP_UNREACH_NLRI containing no NLRI

For any other UPDATE with attributes but no reachable NLRI:
- If any error other than "attribute discard" is encountered
- MUST use "session reset" (cannot confirm NLRI parsed correctly)

### Typed NLRI Address Families (Section 5.4)

For AFI/SAFIs with typed NLRI (MCAST-VPN, MCAST-VPLS, EVPN):
- Unrecognized NLRI types: discard the route
- Do NOT reset session for unknown types

### Attributes Not Discussed

These attributes already have RFC 7606-consistent error handling:
- AS4_PATH (17)
- AS4_AGGREGATOR (18)
- PMSI_TUNNEL (22)
- Tunnel Encapsulation (23)
- AIGP (26)
- PE Distinguisher Labels (27)
- BGP-LS Attribute (29)

These attributes are deprecated:
- DPA (11)
- ADVERTISER (12)
- RCID_PATH/CLUSTER_ID (13)
- SAFI Specific Attribute (19)
- Connector Attribute (20)
- AS_PATHLIMIT (21)
- BGP Entropy Label Capability (28)

## Operational Considerations (Section 6)

### IBGP Inconsistency Risk

Using treat-as-withdraw on IBGP sessions can cause:
- Inconsistent routing within AS
- Forwarding loops
- Black holes

**Mitigation:** Trace malformed attributes to ingress router, apply filter there.

### Debugging Requirements

Implementation MUST provide debugging facilities:
- Log error with affected NLRI
- Log entire malformed UPDATE message
- Enable root cause analysis

## Pitfalls

### Edge Cases

1. **Attribute at end of UPDATE**: If parsing the last attribute would exceed Total Attribute Length, use treat-as-withdraw but rely on Total Attribute Length to find NLRI start

2. **Partial attribute header**: If < 3 bytes (or < 4 with Extended Length) remain, treat-as-withdraw

3. **Zero-length attributes**: Only AS_PATH and ATOMIC_AGGREGATE may have zero length; all others are malformed

4. **EBGP attributes on IBGP**: LOCAL_PREF, ORIGINATOR_ID, CLUSTER_LIST have different handling based on session type

5. **Next Hop length mismatch**: If MP_REACH Next Hop length is wrong, NLRI cannot be located - must session reset or AFI/SAFI disable

### Interop

- Old implementations may send MP_REACH/MP_UNREACH in any position
- Old implementations may send multiple NLRI-bearing fields
- Must accept these but SHOULD NOT send them

### Security

- Malformed optional transitive attributes can be "tunneled" through non-recognizing routers
- Fan-out amplifies impact
- Treat-as-withdraw limits blast radius vs session reset
