---
name: Confirm before switching work
description: Do not change line of work without explicit user confirmation
type: feedback
---

Do not switch to a different line of work without confirming with the user first.

**Why:** In a session starting with "complete the spec", after closing the spec I switched to reviewing/fixing unrelated uncommitted OTC work without asking. The session grew organically from spec closure into deep review into RFC analysis into fixing, with no explicit agreement on the pivot. The user expected a focused session.

**How to apply:** When the original task is done (e.g., spec closed), stop and ask "What next?" instead of picking up other uncommitted work. "Continue what you were doing" means the stated goal, not "find more things to do."
