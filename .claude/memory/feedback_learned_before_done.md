Never present work as complete or ask about committing until the learned summary in `plan/learned/` is written.

**Why:** The implementation was presented as "done" with an Executive Summary but the learned summary was forgotten. The user had to ask. The learned doc is part of completion, not a follow-up.

**How to apply:** In the /implement flow, write the learned summary as part of the final phase, before presenting any summary to the user. The spec's Completion Checklist already says this -- follow it.
