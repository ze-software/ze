**Purpose:** Defines subcodes for the BGP Cease NOTIFICATION message (Error Code 6) to provide more specific reasons for session teardown.

## Subcodes

| Subcode | Name | Description |
|---------|------|-------------|
| 1 | Maximum Number of Prefixes Reached | Peer exceeded configured prefix limit |
| 2 | Administrative Shutdown | Operator-initiated shutdown (RFC 9003 adds message) |
| 3 | Peer De-configured | Peer removed from configuration |
| 4 | Administrative Reset | Operator-initiated reset (RFC 9003 adds message) |
| 5 | Connection Rejected | Inbound connection refused |
| 6 | Other Configuration Change | Non-specific config change |
| 7 | Connection Collision Resolution | RFC 4271 Section 6.8 |
| 8 | Out of Resources | System resource exhaustion |

## Ze Implementation

### Subcode 1: Maximum Number of Prefixes Reached

Ze enforces per-peer per-family prefix maximums. When a peer exceeds the configured maximum for any family:

- NOTIFICATION sent with Error Code 6 (Cease), Subcode 1
- Data field: AFI (2 bytes big-endian) + SAFI (1 byte) + prefix count (4 bytes big-endian)
- Session torn down (when `teardown true`, the default)
- With `teardown false`: session stays up, excess NLRIs dropped silently

Prefix counting: wire-level counting of NLRI entries per UPDATE. Withdrawals processed before announces in the same UPDATE to prevent false triggers from prefix replacement.

### Subcodes 2, 4: Administrative Shutdown/Reset

Ze supports RFC 9003 shutdown communication messages (up to 128 bytes sent, up to 255 bytes accepted).
