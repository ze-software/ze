# RFC 9069 -- Support for Local RIB in BGP Monitoring Protocol (BMP)

## Purpose

Extends BMP (RFC 7854) to support monitoring of the Loc-RIB (local
routing table after best-path selection). Enables collectors to see
the router's actual forwarding decisions, not just what peers
advertise.

## Changes to RFC 7854

### Peer Type 3 (Loc-RIB)

Per-Peer Header Peer Type field value 3 identifies Loc-RIB Route
Monitoring messages. Loc-RIB messages differ from Adj-RIB:

| Field | Adj-RIB (types 0-2) | Loc-RIB (type 3) |
|-------|---------------------|-------------------|
| Peer Address | Remote peer IP | 0 (not applicable) |
| Peer AS | Remote AS number | 0 (not applicable) |
| Peer BGP ID | Remote router ID | Local router ID |
| Peer Distinguisher | RD for VRF | Instance identifier |

### F Flag (Flags bit 7)

For Loc-RIB (Peer Type 3), the V flag position (bit 7) is reused
as the F flag:

| Value | Meaning |
|-------|---------|
| 0 | Route is in Loc-RIB (best path) |
| 1 | Route was filtered by policy (not in Loc-RIB) |

The F flag allows monitoring of routes rejected by import policy,
which never enter the Loc-RIB.

### Affected Message Types

- **Route Monitoring (type 0)**: carries Loc-RIB routes when Peer Type=3
- **Peer Up (type 3)**: announces Loc-RIB monitoring capability; sent/received OPENs are empty (zero-length)
- **Peer Down (type 2)**: signals end of Loc-RIB monitoring
- **Statistics Report (type 1)**: Loc-RIB route counts

### Route Monitoring Content

Loc-RIB Route Monitoring carries BGP UPDATE messages representing
best-path changes:

- **Add/Update**: UPDATE with NLRI and path attributes (ORIGIN, AS_PATH, NEXT_HOP)
- **Withdraw**: UPDATE with withdrawn routes

AS_PATH may be empty for locally originated routes. NEXT_HOP
reflects the actual next-hop after best-path selection.

## Key Constraints

- Peer Type 3 MUST NOT have V, L, A, or O flags set (only F is valid)
- Loc-RIB monitoring is per-instance, not per-peer
- Initial dump sends full Loc-RIB contents as Route Monitoring
- Peer Up for Loc-RIB has zero-length OPEN messages

## Ze Implementation

- Peer type constant: `PeerTypeLocRIB` in `internal/component/bgp/plugins/bmp/header.go`
- Phase 4 of spec-bmp-5-sender-compliance (deferrable)
- Best-path changes from RIB plugin via EventBus `(bgp-rib, best-change)`
