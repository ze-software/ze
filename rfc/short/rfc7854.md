# RFC 7854 -- BGP Monitoring Protocol (BMP)

## Purpose

BMP enables routers to stream BGP routing state to external monitoring
stations over TCP. Unidirectional: router to collector. Covers Adj-RIB-In,
peer state changes, and statistics.

## Wire Format

### Common Header (6 bytes, all messages)

| Offset | Size | Field |
|--------|------|-------|
| 0 | 1 | Version (must be 3) |
| 1 | 4 | Message Length (total including header) |
| 5 | 1 | Message Type (0-6) |

### Per-Peer Header (42 bytes, types 0-3 and 6)

| Offset | Size | Field |
|--------|------|-------|
| 0 | 1 | Peer Type: 0=Global, 1=L3VPN, 2=Local, 3=Loc-RIB |
| 1 | 1 | Flags: bit7=V(IPv6), bit6=L(post-policy), bit5=A(2-byte-AS), bit4=O(Adj-RIB-Out) |
| 2 | 8 | Peer Distinguisher (RD for VRF peers) |
| 10 | 16 | Peer Address (IPv4 as ::ffff:x.x.x.x) |
| 26 | 4 | Peer AS (always 4-byte) |
| 30 | 4 | Peer BGP ID |
| 34 | 4 | Timestamp seconds |
| 38 | 4 | Timestamp microseconds |

### TLV Format (4-byte header)

Type (2) + Length (2) + Value (variable).

## Message Types

| Type | Name | Per-Peer Header | Content |
|------|------|-----------------|---------|
| 0 | Route Monitoring | Yes | BGP UPDATE message |
| 1 | Statistics Report | Yes | Count (4) + stat TLVs |
| 2 | Peer Down | Yes | Reason (1) + optional NOTIFICATION/FSM |
| 3 | Peer Up | Yes | Local addr (16) + ports (4) + sent OPEN + recv OPEN + optional TLVs |
| 4 | Initiation | No | TLVs: sysName (2), sysDescr (1), string (0) |
| 5 | Termination | No | TLVs: string (0), reason (1) |
| 6 | Route Mirroring | Yes | TLVs: BGP message (0), messages lost (1) |

## Peer Down Reasons

| Code | Meaning | Data |
|------|---------|------|
| 1 | Local closed, NOTIFICATION sent | BGP NOTIFICATION PDU |
| 2 | Local closed, no NOTIFICATION | 2-byte FSM event code |
| 3 | Remote closed, NOTIFICATION recv | BGP NOTIFICATION PDU |
| 4 | Remote closed, no data | None |
| 5 | Peer de-configured | None |

## Statistics Types (Section 4.8)

Types 0-13 defined. Types 0-6 are 64-bit counters for policy rejections,
duplicates, and loop detection. Types 7-8 are Adj-RIB-In and Loc-RIB route
counts. Types 9-10 are per-AFI/SAFI variants (11-byte: AFI(2)+SAFI(1)+value(8)).
Types 11-13 cover treat-as-withdraw and duplicate updates.

## Session Lifecycle

1. TCP connection established (IANA port 11019)
2. Initiation message with sysName/sysDescr
3. For each established peer: Peer Up with OPEN messages
4. Route Monitoring: initial RIB dump then incremental updates
5. Periodic Statistics Reports
6. Peer Down on session loss
7. Termination on BMP session close

## Reconnection

RFC suggests minimum 30 seconds, maximum 720 seconds between reconnection
attempts with exponential backoff.

## Related RFCs

- RFC 8671: Adj-RIB-Out support (O flag in per-peer header)
- RFC 9069: Local RIB support (peer type 3, F flag)
- RFC 9736: Peer Up TLV namespace clarification
