# RFC 9728 - OAuth 2.0 Protected Resource Metadata

## Meta

| Field | Value |
|-------|-------|
| RFC | 9728 |
| Title | OAuth 2.0 Protected Resource Metadata |
| Status | Proposed Standard |
| Date | April 2025 |
| Authors | A. Parecki, M. Jones |
| Related | RFC 8414 (AS metadata), RFC 8707 (resource indicators), RFC 6750 (Bearer) |

**Purpose:** Defines a discovery document the resource server publishes at a well-known URL so OAuth 2.1 clients can learn which authorization servers issue tokens for that resource, what scopes are recognised, and how Bearer tokens must be presented. The MCP 2025-06-18 authorization profile makes this document a MUST for MCP resource servers.

## Well-Known Endpoint (Section 3.1)

| Item | Value |
|------|-------|
| Path | `/.well-known/oauth-protected-resource` |
| Method | GET |
| Auth | MUST be reachable without authentication |
| Content-Type | `application/json` |

## Metadata Document (Section 2)

| Field | Requirement | Ze implementation |
|-------|-------------|-------------------|
| `resource` | MUST; canonical URL identifying this resource | Operator-configured via `environment.mcp.oauth.audience` (or `metadata-resource` override); NEVER derived from request Host |
| `authorization_servers` | MUST; JSON array of AS issuer URLs | Single entry built from `environment.mcp.oauth.authorization-server` |
| `scopes_supported` | MAY; JSON array of scope names | Mirrors `environment.mcp.oauth.required-scopes` |
| `bearer_methods_supported` | MAY; array of delivery methods | Always `["header"]` -- MCP only accepts `Authorization: Bearer` |
| `resource_documentation` | MAY; URL | Not emitted |
| `resource_signing_alg_values_supported` | MAY | Not emitted; ze validates RS256/RS384/RS512/ES256/ES384 |

## 401 Challenge (Section 5.1)

A resource server MUST name the metadata URL in the WWW-Authenticate header so a client can discover the AS without out-of-band configuration:

```
WWW-Authenticate: Bearer realm="ze-mcp",
                  error="invalid_token",
                  error_description="...",
                  resource_metadata="https://mcp.example/.well-known/oauth-protected-resource"
```

| Section | Requirement |
|---------|-------------|
| 5.1 | 401 response MUST include `WWW-Authenticate: Bearer resource_metadata="<url>"`. The URL MUST be absolute. |
| 5.2 | The `error` / `error_description` parameters use RFC 6750 vocabulary: `invalid_request`, `invalid_token`, `insufficient_scope`. |

## Key Behavior

| Section | Requirement |
|---------|-------------|
| 3.2 | Clients MAY cache the document. `Cache-Control: max-age=N` is allowed; resource servers SHOULD return a short-ish max-age (e.g., 300 s). |
| 2 | Unknown fields in the document MUST be ignored by clients. |
| 3 | The document MUST be served over TLS when the resource itself is reached over TLS; plaintext HTTP is only acceptable on loopback. |

## Ze Implementation

- Handler: `internal/component/mcp/oauth.go` `writeResourceMetadata`
- Serving: `internal/component/mcp/streamable.go` ServeHTTP dispatches GET `/.well-known/oauth-protected-resource` to the handler only when `AuthMode=OAuth`; other modes return 404.
- Resource field: `cfg.OAuth.MetadataResource` with fallback to `cfg.OAuth.Audience`. Never derived from `r.Host` -- a reverse proxy cannot change the resource's advertised identity.
