# RFC 2866 - RADIUS Accounting

## Meta

| Field | Value |
|-------|-------|
| RFC | 2866 |
| Title | RADIUS Accounting |
| Status | Informational |
| Date | June 2000 |
| Author | C. Rigney |
| Obsoletes | RFC 2139 |
| Updates | - |
| Updated by | RFC 5080 |

**Purpose:** Defines the RADIUS Accounting protocol for carrying
accounting information between a NAS and a shared accounting server.
Uses the same packet format as RADIUS Authentication (RFC 2865) on
UDP port 1813.

**Scope:** Accounting is fire-and-forget from the session's perspective:
accounting failures MUST NOT tear down user sessions. The NAS sends
Accounting-Request packets; the server responds with Accounting-Response.

## Packet Format (Section 3)

Same as RFC 2865 with these codes:
- **Code 4:** Accounting-Request
- **Code 5:** Accounting-Response

## Request Authenticator (Section 3)

Unlike authentication, the Accounting-Request authenticator is computed,
not random:

```
RequestAuth = MD5(Code + ID + Length + 16_zero_octets + Attributes + Secret)
```

The 16 zero octets replace where the authenticator will be placed.

## Key Attributes (Section 5)

| Type | Name | Format | Notes |
|------|------|--------|-------|
| 40 | Acct-Status-Type | Integer | 1=Start, 2=Stop, 3=Interim-Update |
| 41 | Acct-Delay-Time | Integer | Seconds since event |
| 42 | Acct-Input-Octets | Integer | Bytes received from user |
| 43 | Acct-Output-Octets | Integer | Bytes sent to user |
| 44 | Acct-Session-Id | String | Unique per session |
| 46 | Acct-Session-Time | Integer | Seconds session has been active |
| 47 | Acct-Input-Packets | Integer | Packets received from user |
| 48 | Acct-Output-Packets | Integer | Packets sent to user |

## Accounting Lifecycle

1. **Start (Status-Type=1):** Sent when user session begins (NCP completes).
   Includes User-Name, NAS-IP-Address, Acct-Session-Id, Service-Type.

2. **Interim-Update (Status-Type=3):** Sent periodically during active session.
   Includes cumulative counters (octets, packets, session time).

3. **Stop (Status-Type=2):** Sent when session ends.
   Includes final duration, byte/packet counters, termination cause.

## Acct-Session-Id (Section 5.5)

MUST be unique across all active sessions on the NAS. The NAS generates
it; the server uses it to correlate Start/Interim/Stop records for the
same session.

## Implementation Constraints

- Accounting failures must not affect session state.
- Interim-Update interval is locally configured (typically 300s).
- Acct-Session-Id must be unique per NAS per session.
- Same retransmit and failover rules as RFC 2865.
