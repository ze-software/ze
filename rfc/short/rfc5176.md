# RFC 5176 - Dynamic Authorization Extensions to RADIUS

## Meta

| Field | Value |

| Status | Informational |

**Purpose:** Extends RADIUS to allow a RADIUS server to initiate
changes to active sessions on the NAS. Two new message types:
Change-of-Authorization (CoA) for modifying session parameters, and
Disconnect-Message (DM) for terminating sessions.

**Scope:** The NAS listens on UDP port 3799 for incoming requests from
the RADIUS server. This reverses the normal client-server direction:
the RADIUS server sends requests, and the NAS responds. The same
shared secret, authenticator computation, and TLV attribute encoding
from RFC 2865 apply.

## Packet Codes (Section 3)

| Code | Name | Direction |
|------|------|-----------|
| 40 | Disconnect-Request | Server -> NAS |
| 41 | Disconnect-ACK | NAS -> Server |
| 42 | Disconnect-NAK | NAS -> Server |
| 43 | CoA-Request | Server -> NAS |
| 44 | CoA-ACK | NAS -> Server |
| 45 | CoA-NAK | NAS -> Server |

## Request Authenticator (Section 3.5)

The Request Authenticator in CoA-Request and Disconnect-Request is
computed the same way as Accounting-Request (RFC 2866 Section 3):

```
MD5(Code + Identifier + Length + 16-zero-octets + Attributes + Secret)
```

The NAS MUST verify the Request Authenticator before processing any
attributes. Packets with invalid authenticators MUST be silently
discarded.

## Response Authenticator (Section 3.5)

Response packets (ACK/NAK) use the standard Response Authenticator
from RFC 2865 Section 3:

```
MD5(Code + Identifier + Length + Request-Authenticator + Attributes + Secret)
```

## Session Identification (Section 3.3)

The RADIUS server identifies the target session using one or more of:

| Attribute | Type | Purpose |
|-----------|------|---------|
| Acct-Session-Id (44) | String | Unique session identifier from Accounting-Start |
| User-Name (1) | String | PPP authenticated identity |
| NAS-Port (5) | Integer | Port number on the NAS |
| Calling-Station-Id (31) | String | Subscriber phone/MAC |
| NAS-IP-Address (4) | IPv4 | NAS identification |
| NAS-Identifier (32) | String | NAS identification |

Multiple identification attributes narrow the match. At minimum, one
session identification attribute MUST be present.

## CoA Use Cases (Section 2)

- **Bandwidth change:** Update Filter-Id or vendor-specific rate
  attributes on an active session. The NAS applies new TC/QoS rules
  without disconnecting the subscriber.

- **Session disconnect:** Force-terminate a session (e.g., prepaid
  credit exhausted, administrative action).

- **Authorization change:** Modify any RADIUS-provisioned session
  attribute (VLAN, ACL, idle timeout).

## Ze Implementation Notes

- NAS listens on configurable UDP port (default 3799)
- Shared secret reused from the RADIUS auth/acct server config
- Session identification by Acct-Session-Id (primary) or User-Name + NAS-Port (fallback)
- CoA with Filter-Id containing rate string (e.g. "10mbit") triggers SessionRateChange EventBus event
- Disconnect-Request triggers TeardownSession on the L2TP subsystem
- Source address filtering recommended (accept only from configured RADIUS servers)
