# RFC 9514: BGP-LS Extensions for Segment Routing over IPv6 (SRv6)

## Meta

| Field | Value |
|-------|-------|
| RFC | 9514 |
| Title | Border Gateway Protocol - Link State (BGP-LS) Extensions for Segment Routing over IPv6 (SRv6) |
| Status | Standards Track |
| Date | December 2023 |
| Obsoletes | None |
| Updates | None |
| Obsoleted-by | None |

**Purpose:** Defines extensions to BGP-LS to advertise SRv6 segments along with their behaviors and other attributes via BGP. Analogous to RFC 9085 for SR-MPLS.

**Scope:** BGP-LS (AFI/SAFI 16388/71) extensions for SRv6 topology distribution.

**Key Dependencies:**
- RFC 7752: BGP-LS base specification
- RFC 8402: Segment Routing Architecture
- RFC 8986: SRv6 Network Programming
- RFC 9085: BGP-LS Extensions for SR (MPLS)
- RFC 9086: BGP-LS Extensions for BGP-EPE
- RFC 9352: IS-IS Extensions for SRv6
- RFC 9513: OSPFv3 Extensions for SRv6

### IGP to BGP-LS Field Copying

- Flags: Copy directly from IS-IS sub-TLV or OSPFv3 sub-TLV
- Algorithm: Copy from source
- Weight: Copy from source
- SID: Copy 128-bit value
- Endpoint Behavior: Copy code point

## MUST Requirements

### Tx (Sender)

- "A single instance of this TLV MUST be included in the BGP-LS Attribute for each SRv6-capable node." (Section 3.1)
- "This field MUST contain a single SRv6 SID Information TLV (Section 6.1)" (Section 6)
- "The SRv6 Endpoint Behavior TLV is a mandatory TLV that MUST be included in the BGP-LS Attribute associated with the BGP-LS SRv6 SID NLRI." (Section 7.1)
- "This TLV MUST be included along with SRv6 SIDs that are associated with the BGP PeerNode or PeerSet functionality." (Section 7.2)
- Reserved fields: "MUST be set to 0 when originated" (Sections 3.1, 4.1, 4.2, 5.1, 7.1, 7.2)
- Undefined flags: "MUST be set to 0 when originating" (Section 7.1)
- Algorithm: "value MUST be 0 unless an algorithm is associated locally with the SRv6 Locator" (Section 7.1)
- SID Structure sum: "MUST be less than or equal to 128" (Section 8)

### Rx (Receiver)

- Reserved fields: "ignored on receipt" (Sections 3.1, 4.1, 4.2, 5.1, 7.1, 7.2)
- Undefined flags: "ignored on receipt" (Section 7.1)

## SHOULD/MAY

- [MAY] "MAY contain the Multi-Topology Identifier TLV" - in SRv6 SID Descriptors (Section 6)
- [MAY] "MAY be assigned to one or more End.X SIDs" - S-Flag PeerSet SID (Section 7.2)
- [SHOULD] "isolation of BGP-LS peering sessions is RECOMMENDED to ensure that BGP-LS topology information (including the newly added SR information) is not advertised to an external BGP peering session outside the SR domain" - security (Section 11)

## Pitfalls

### Edge Cases

- **Prefix NLRI dual meaning:** Prefix NLRI with SRv6 Locator TLV but without IGP Metric TLV (1095) is ONLY a locator advertisement, not a routing prefix
- **LAN End.X Neighbor ID size:** IS-IS uses 6 octets (System-ID), OSPFv3 uses 4 octets (Router-ID) - different TLV types required
- **PeerSet vs PeerNode:** PeerSet has multiple SRv6 BGP PeerNode SID TLVs with S-Flag set; PeerNode has exactly one without S-Flag

### Interop

- **SR-MPLS vs SRv6 EPE signaling difference:** SR-MPLS uses additional Link NLRI per peering session for PeerNode/PeerSet SIDs; SRv6 uses SRv6 SID NLRI with peer info as attributes (see Appendix A)
- **IGP flag semantics:** Flags copied from IS-IS and OSPFv3 may have different bit definitions - check source RFC

### Security

- **Trusted domain:** BGP-LS SRv6 extensions operate within SR trusted domain (RFC 8402)
- **Isolation:** BGP-LS sessions SHOULD be isolated from external BGP peers to prevent SRv6 topology leakage
- **Authentication:** Relies on underlying IGP/BGP security mechanisms

## Compatibility

### Non-Supporting Peers

- Standard BGP-LS unknown TLV handling applies (RFC 7752)
- New NLRI type 6 will be treated as unknown by peers not supporting this RFC
- Semantic validation left to BGP-LS consumers (applications/controllers), not BGP itself

### Feature Negotiation

- No explicit capability negotiation defined
- Detection via presence of SRv6 Capabilities TLV in Node NLRI
- Algorithm support via SR-Algorithm TLV (RFC 9085)
- MSD limits via Node/Link MSD TLV (RFC 8814)
