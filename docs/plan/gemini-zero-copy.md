# Zero-Copy Analysis & Optimization Opportunities

**Date:** 2026-01-04
**Focus:** Memory Allocations & Data Copying in Hot Paths

## Executive Summary
This analysis identifies specific areas in the ZeBGP codebase where data copying occurs unnecessarily. Given Go's slice mechanics, several "hot paths" (particularly in message chunking and splitting) are currently performing O(N) copies where O(1) sub-slicing would be sufficient. Addressing these will significantly improve throughput and reduce GC pressure.

## 1. Critical: Unnecessary Copying in NLRI Chunking
**File:** `pkg/bgp/message/chunk_mp_nlri.go`
**Function:** `ChunkMPNLRI`

### Current Behavior
The function iterates through the input `nlri` byte slice and appends each NLRI to a `current` accumulator slice using `append(...)`.
```go
// Current implementation
current = append(current, nlri[offset:offset+nlriSize]...)
```
This causes Go to allocate a new backing array for `current` and copy bytes from the input `nlri` slice every time.

### Optimization Opportunity
Since the input `nlri` slice usually represents a contiguous block of data (either from an immutable `Route` cache or a constructed buffer), we can treat it as read-only and return **sub-slices** instead of copying.
```go
// Optimized approach (Zero-Copy)
// Track start/end indices and slice the original array
chunks = append(chunks, nlri[chunkStart:offset])
```
**Impact:** Eliminates data copying for the entire NLRI payload during splitting.

## 2. High Impact: Attribute Removal Allocation
**File:** `pkg/bgp/message/update_split.go`
**Function:** `removeAttribute` (called by `splitUpdateWithMP`)

### Current Behavior
To remove MP_REACH/MP_UNREACH attributes from the attribute set, the code creates a *new* slice and copies all other bytes into it.
```go
result := make([]byte, 0, len(pathAttrs)-(end-start))
result = append(result, pathAttrs[:start]...)
result = append(result, pathAttrs[end:]...)
```
This logic runs twice if both attributes are present, resulting in two full allocations and copies of the path attributes for *every* split UPDATE.

### Optimization Opportunity
1.  **Composite Writer:** Instead of creating a new `baseAttrs` slice, construct the final message using a `net.Buffers` style approach (vectorized I/O) or a gather-list that points to: `[Header, AttrsBeforeMP, AttrsAfterMP, NewMPAttribute]`.
2.  **In-Place Modification (Careful):** If the `pathAttrs` buffer ownership allows (e.g. it was just generated by a builder), we could shift bytes in place, though this is riskier. The Composite Writer approach is safer and zero-copy.

## 3. Medium Impact: Update Builder Intermediate Slices
**File:** `pkg/bgp/message/update_build.go`
**Function:** `Build*` methods (e.g., `BuildUnicast`)

### Current Behavior
The builders allocate a temporary slice of interfaces `[]attribute.Attribute` to hold attributes before packing them.
```go
var attrs []attribute.Attribute
attrs = append(attrs, ...)
// ...
attrBytes := attribute.PackAttributesOrdered(attrs)
```
This creates unnecessary heap allocations for the slice and potential interface boxing.

### Optimization Opportunity
Implement a **Direct-to-Wire Builder**.
Instead of collecting objects, writing directly to a pre-sized byte buffer (or a reused buffer pool) avoids the intermediate slice entirely.
`ub.WriteOrigin(...)`, `ub.WriteASPath(...)` -> writes bytes immediately.

## 4. Medium Impact: Raw Attribute Packing
**File:** `pkg/reactor/peer.go`
**Function:** `packRawAttribute`

### Current Behavior
Every call allocates a new byte slice.
```go
buf := make([]byte, 3+valueLen)
// copy...
return buf
```
This is called for every raw attribute in every static route update.

### Optimization Opportunity
Pass a reused buffer/builder to this function: `packRawAttribute(buf *bytes.Buffer, ra RawAttribute)`. This allows the caller to amortize allocations over many attributes.

## Summary of Proposed Changes

| Component | Current State | Proposed Zero-Copy State | Est. Improvement |
|-----------|---------------|--------------------------|------------------|
| `ChunkMPNLRI` | `append` (Alloc+Copy) | Sub-slicing (Pointer arithmetic) | **High** |
| `splitUpdateWithMP` | `make` + `copy` (x2) | Scatter-Gather slice list | **High** |
| `UpdateBuilder` | `[]Attribute` allocs | Direct byte writing | Medium |
| `packRawAttribute` | Per-call `make` | Buffer pooling | Low/Medium |
