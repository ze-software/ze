# Ze Recap

Use this when the user wants quick orientation rather than a full plan.

## Workflow

1. Read the selected spec and its metadata if one is active.
2. Read the latest per-spec session state file if it exists.
3. Summarize git state with current branch, uncommitted changes, and recent commits.
4. Check formal deferrals and any obvious skipped items from the current session.
5. Present five short sections: goal, done, in progress, remaining, deferred, and skipped.

## Rules

- Read-only.
- Do not turn the recap into a new plan.
